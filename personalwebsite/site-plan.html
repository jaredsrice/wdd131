Jared Rice

WDD 131

Overview

Purpose

A simple website that helps users track monthly income and expenses. It allows users to add transactions, categorize spending, and view a clear summary of their financial activity. The goal is to make personal budgeting easy and beginner-friendly.

Audience

The site is designed for anyone who wants a basic and clean budgeting tool without using complicated software.

Dynamic elements

When someone adds a new income or expense, the page will update right away. I am storing all transactions in an array of objects, and using functions to add entries, calculate totals, and filter by category. JavaScript will listen for button clicks and form submissions and then update the DOM to show the new values. Amounts will be color coded based on whether they are income or expenses, and the summary card on the home page will update during the same session based on the data in the transactions list.

Content

Home page

I have a few images that I'll be using to add a bit of flair to my page, but there's a lot of them and I don't want to flood the site plan with them. In terms of content, there will be initial placeholder values in the 'Summary Card', which is the top one. It'll be a quick overview of total income, total expenses, and current balance. The bottom sections will have weekly spend values, amount of spending categories, and a percentage savings goal.

The transactions page will start with several placeholder values so the table is not empty, and will be a mix of income and expenses to demonstrate functionality. Wireframes

Home

The software I was using here made spacing difficult, but this is a basic layout of what I want the home page to look like. The images will be either my logo or opensource icons that fit the theme of budgeting and finance. I'm also not doing a traditional navigation bar, it didn't really fit with what I'm going for with the design. The values in the cards will be partially dynamic, updating based on what the user inputs on the second page as long as both pages are open in the same session. I'll put initial placeholder values in so the user can see what it will look like.

[Page 2]

The big middle box with the 'X' is where the dynamic budget table will go. The user will be able to add income and expenses using the form at the bottom, and the table will update automatically to show their current expenses. It will have 4 columns for Name, Amount, Category, and Date. The amount values will be color coded to show income in green and expenses in red, with + and - symbols respectively. The categories will be labeled and color coded as well. A disclamer, if anything is too complicated I probably will simplify or remove it. The bottom form won't have images under Transaction name or amount, but are text boxes.
